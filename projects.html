6. Data Engineer project

I've built a comprehensive ETL (Extract, Transform, Load) pipeline using AWS services to process data from the Spotify API. This project leverages several AWS technologies to extract data, transform it into a structured format, and make it ready for analysis. Hereâ€™s a quick overview:

Project Overview

  • Extract:
    • Spotify API: Source of raw data.
    • AWS Lambda (Data Extraction): Extracts data from the Spotify API every minute, triggered by Amazon CloudWatch.
    • Amazon S3 (Raw Data): Stores the extracted raw data.
  • Transform:
    • AWS Lambda (Data Transformation): Processes and transforms the raw data, saving it into a structured format in S3.
    • Amazon S3 (Transformed Data): Stores the transformed data ready for analysis.
  • Load:
    • AWS Glue Crawler: Infers the schema of the transformed data and updates the AWS Glue Data Catalog.
    • AWS Glue Data Catalog: Stores the schema and metadata.
    • Amazon Athena: Allows SQL querying and analysis of the transformed data.

Technologies Used

  • AWS Lambda for serverless compute
  • Amazon S3 for data storage
  • Amazon CloudWatch for triggering and monitoring
  • AWS Glue for data cataloging and schema inference
  • Amazon Athena for data analysis

Key Features

  • Automated data extraction from Spotify API
  • Real-time data transformation and storage in S3
  • Schema inference and cataloging with AWS Glue
  • SQL-based data analysis using Amazon Athena

How It Works

  • Data Extraction: A CloudWatch rule triggers the data extraction Lambda function, which fetches data from the Spotify API and stores it in S3.
  • Data Transformation: An S3 event triggers the data transformation Lambda function, which processes the raw data, saves the transformed data, and moves the processed files to avoid reprocessing.
  • Data Loading: An AWS Glue Crawler updates the Data Catalog with the schema of the transformed data, making it available for querying in Athena.
